compare:

Compares two NCDFDimension object's variable ID. 

-(
        NSComparisonResult)compare:(
        id)object;
Discussion

This method can be used to sort NCDFDimension objects using their variable IDs. Returns NSOrderedAccending,NSOrderedDecending, or NSOrderedSame.

dimensionID

Access the dimension id. 

-(
        int)dimensionID;
Discussion

Dimension ID numbers are how variables define the shape of the datasets. Thus a variable will refer to a series of dimension ID numbers representing each dimension in the dataset shape. Hence, a caller cannot set the dimension ID number.

VALIDATION NOTES: Tested extensively and appears to function as expected.

initWithFileName:dimID:name:length:handle:

Initializes a new NCDFDimension for a dimension within a file. 

-(
        id)initWithFileName:(
        NSString *)thePath dimID:(
        int)number name:(
        NSString *)name length:(
        size_t)length handle:(
        NCDFHandle *)handle;
Parameter Descriptions
thePath
the path to an existing netcdf file.
number
the dimension identification number. This should be handled by NCDFHandle.
name
NSString with the dimension name. If the string is not consistent with netcdf naming rules, it will be parsed.
length
The length of the dimension in units.
handle
The owning NCDFHandle.
Discussion

This is the standard method for creating a NCDFDimension from a dimension in an existing file. This method should only be invoked by a parent NCDFHandle.

propertyList

Returns a NSDictionary property list. 

-(
        NSDictionary *)propertyList;
Discussion

Returns a NSDictionary representation that can contains only propertyList compatible objects. A property list will have the following items: Fields:

NCDFDimensionPropertyListFieldFileName

NCDFDimensionPropertyListFieldDimensionID

NCDFDimensionPropertyListFieldDimensionName

NCDFDimensionPropertyListFieldLength



Note: Using this method to create a new dimension in a file can have unexpected results when working with unlimited dimensions. Since length represents the actual length of a dimension, using a property list to create the same dimension elsewhere will cause the dimension to be LIMITED in length. Reset to 0 if unlimited.

In future versions, a new field will be added to determine if unlimited.



VALIDATION NOTES: Tested extensively and appears to function as expected.

setDimLength:

Set the dimension dimLength. 

-(
        void)setDimLength:(
        size_t)newLength;
Parameter Descriptions
newLength
Length in units.
Discussion

This method will change the length of the dimension. In some cases, it may be possible to set the dimension as unlimited by setting the length to 0. In this case, no unlimited dimensions can exist in the file prior to setting a dimension to unlimited length.
